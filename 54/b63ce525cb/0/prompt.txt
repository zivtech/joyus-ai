**[HIGH] InMemoryEventBus.publish does not isolate handler errors -- one failing handler prevents others from running**
File: `/Users/AlexUA/claude/joyus-ai/.worktrees/009-automated-pipelines-framework-WP02/joyus-ai-mcp-server/src/pipelines/event-bus/interface.ts:76-82`
Issue: `publish()` uses `Promise.all(dispatches)` which will reject as soon as any single handler rejects. This means (a) the remaining handler promises are still running but their results are ignored, and (b) the `publish` call itself throws, potentially crashing the caller. `PgNotifyBus` correctly wraps each handler in try/catch (line 111-115), but `InMemoryEventBus` does not. ```typescript
for (const sub of this.subscriptions.values()) {
