### 1b. `requireSession` — Session cookie auth (Express middleware)
Checks `req.session?.userId`, returns 401 JSON for API routes. Replaces manual checks in:
- `scheduler/routes.ts` lines 24, 252, 298, 325, 374, 465, 524 (~8 instances)

### 1c. `requireSessionOrRedirect` — Session cookie with redirect (Express middleware)
Same as `requireSession` but redirects to `/auth` for UI routes. Replaces manual checks in:
- `auth/routes.ts` lines 52-54, 343, 451, 547, 648
- `scheduler/routes.ts` lines 24, 373

**Pattern:** Follow the factory style from `content/mediation/auth.ts` — no DB import at module level, consistent error responses.
