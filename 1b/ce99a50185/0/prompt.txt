# Issue #5: Persist Export Download Tokens to Database

## Context

GitHub issue #5 (grndlvl): `exports/service.ts` stores export jobs and download tokens in in-memory JavaScript `Map`s. Any server restart wipes active download links, breaking the 15-minute TTL promise to users. Solution: persist to PostgreSQL via Drizzle ORM.

**Current state:** Two Maps in `service.ts`:
- `exportJobs: Map<string, ExcelExportJob>` — job records
- `downloadTokenToJob: Map<string, { jobId: string; expiresAtMs: number }>` — token→job lookup

**Existing patterns to follow:**
- Tables defined in `src/db/schema.ts` (public namespace) with pgEnum, pgTable, index helpers
- CUID2 IDs via `createId()`, timestamps with `defaultNow()`, `$inferSelect`/`$inferInsert` type exports
- Relations defined alongside tables; unified in `db/client.ts` via `* as schema`

## Task 2: Rewrite `service.ts` to use database

**Edit** `joyus-ai-mcp-server/src/exports/service.ts`

Remove both `Map` declarations. Replace with Drizzle queries:

| Function | Map operation → DB operation |
|---|---|
| `createExcelExportJob()` | `exportJobs.set()` → `db.insert(exportJobs).values({...}).returning()` |
| `getExcelExportJobForUser()` | `exportJobs.get()` → `db.select()...where(eq(id) AND eq(userId) AND eq(tenantId))` |
| `resolveDownloadToken()` | `downloadTokenToJob.get()` + manual expiry → `db.select()...where(eq(downloadToken) AND gt(downloadExpiresAt, now()))` |

Excel workbook generation logic stays unchanged — only the storage layer changes.

Add `cleanupExpiredExports()` for future scheduler use:
```typescript
export async function cleanupExpiredExports(): Promise<number> {
  const result = await db.delete(exportJobs)
    .where(and(isNotNull(exportJobs.downloadExpiresAt), lt(exportJobs.downloadExpiresAt, sql`now()`)))
    .returning({ id: exportJobs.id });
  return result.length;
}
```

Don't wire to scheduler yet — just export the function.
