## Task 1: Create shared auth middleware

**New file:** `src/auth/middleware.ts`

Three middleware functions covering the two auth mechanisms currently duplicated:

### 1a. `requireBearerToken` ‚Äî MCP token auth (Express middleware)
Extracts `Authorization: Bearer <token>`, calls existing `getUserFromToken()` from `verify.ts`, attaches user to `req.mcpUser`. Replaces inline auth at:
- `index.ts:170-179` (`/mcp` POST)
- `index.ts:273-281` (`/mcp/sse` GET ‚Äî token from query param, so this variant takes token from either header or `?token=`)

### 1b. `requireSession` ‚Äî Session cookie auth (Express middleware)
Checks `req.session?.userId`, returns 401 JSON for API routes. Replaces manual checks in:
- `scheduler/routes.ts` lines 24, 252, 298, 325, 374, 465, 524 (~8 instances)

### 1c. `requireSessionOrRedirect` ‚Äî Session cookie with redirect (Express middleware)
Same as `requireSession` but redirects to `/auth` for UI routes. Replaces manual checks in:
- `auth/routes.ts` lines 52-54, 343, 451, 547, 648
- `scheduler/routes.ts` lines 24, 373

**Pattern:** Follow the factory style from `content/mediation/auth.ts` ‚Äî no DB import at module level, consistent error responses.

**Type augmentation:** Extend `Express.Request` in existing `types/express-session.d.ts` to add `mcpUser?: UserWithConnections`.

### Files modified:
- **Create** `src/auth/middleware.ts` ‚Äî the three middleware functions
- **Edit** `src/types/express-session.d.ts` ‚Äî add `mcpUser` to Request
